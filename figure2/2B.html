Get Y¶

for each feature if day == count, then, add feature value for that day to sum. If day day != count then take average of sum and append to feature_y. Also update count to new day. Finally, append feature_y to all_features_y, which is a matrix with the average value for each feature per day. Similar to df.GroupBy(["Days"]).mean().

Get unique X (pre sorted for days_after_birth)¶

for every value in column "Days_after_birth" if value != count then append value to x and update count to value. X already has 0 in beginning.

    feature_y = []
    count = 0
    sum_ = []

            sum_.append(df.iloc[m,f])
        else:

Quality filtering and host removal¶

347 and 60 demultiplexed fastq files from the Born-immune and IMPRINT cohort respectively were downstreams processed using the same pipeline and parameters. Demultiplexed samples were quality filtered using fastp v0.20.0 (https://doi.org/10.1093/bioinformatics/bty560), and host contamination removed using Kraken v2.0.8_beta (https://doi.org/10.1186/s13059-019-1891-0) by mapping against the NCBI's GRCh38.p13 database (https://www.ncbi.nlm.nih.gov/assembly/GCF_000001405.39). Both steps were run using default settings in StaG-mwc v.0.4.1 (https://doi.org/10.5281/zenodo.1483891).

Taxonomic and functional profiling¶

Taxonomic profiles were established using MetaPhlAn v3.0.5 (https://www.biorxiv.org/content/10.1101/2020.11.19.388223v1) and functional HMO profiles generated with HUMAnN2 v.2.8.1 (https://www.ncbi.nlm.nih.gov/pmc/articles/PMC6235447/) bypassing all steps except "nucleotide-search" and "evalue 0.00001" with a customized nucleotide database of HMO genes (fig X) instead of the chochophlan database. Both taxonomic and functional profiling were incorporated into StaG-mwc.
